   49: =head1 DESCRIPTION
   50: 
   51: Encode::Wide provides functions for converting wide (Unicode) characters into ASCII-safe
   52: formats suitable for embedding in HTML or XML documents.
   53: It is especially useful when dealing with text containing accented or typographic characters that need
   54: to be safely represented in markup.
   55: 
   56: Other modules exist to do this,
   57: however they tend to have assumptions on the input,
   58: whereas this should work with UTF-8, Unicode, or anything that's common.
   59: 
   60: The module offers two exportable functions:
   61: 
   62: =over 4
   63: 
   64: =item * C<wide_to_html(string => $text)>
   65: 
   66: Converts all non-ASCII characters in the input string to their named HTML entities if available,
   67: or hexadecimal numeric entities otherwise.
   68: Common characters such as `Ã©`, `Ã `, `&`, `<`, `>` are
   69: converted to their standard HTML representations like `&eacute;`, `&agrave;`, `&amp;`, etc.
   70: 
   71: =item * C<wide_to_xml(string => $text)>
   72: 
   73: Converts all non-ASCII characters in the input string to hexadecimal numeric entities.
   74: Unlike HTML, XML does not support many named entities, so this function ensures compliance
   75: by using numeric representations such as `&#xE9;` for `Ã©`.

  106: =head2 wide_to_html
  107: 
  108: =head3 Purpose
  109: 
  110: Convert a Unicode / UTF-8 string into a pure-ASCII HTML fragment by replacing
  111: every non-ASCII character with its named HTML entity (e.g. C<&eacute;>) where
  112: one exists, or a hexadecimal numeric entity (e.g. C<&#xNNNN;>) otherwise.
  113: Bare ampersands, angle brackets, and double-quotes are also escaped so the
  114: result is safe to embed in an HTML attribute or body.
  115: 
  116: =head3 Arguments
  117: 
  118: Named parameters (or a single positional string as the first argument):
  119: 
  120: =over 4
  121: 
  122: =item C<string> (required)
  123: 
  124: The text to encode.  May be a plain scalar or a reference to a scalar.
  125: Must be defined; passing C<undef> causes the function to C<die>.
  126: 
  127: =item C<keep_hrefs> (optional, default 0)
  128: 
  129: When true, C<< < >>, C<< > >>, and C<< " >> are B<not> escaped,
  130: so that literal HTML hyperlinks embedded in the input survive intact.
  131: 
  132: =item C<keep_apos> (optional, default 0)
  133: 
  134: When true, apostrophes (C<'>) are B<not> converted to C<&apos;>.
  135: Useful when the result will appear inside a JavaScript string literal.
  136: 
  137: =item C<complain> (optional)
  138: 
  139: A code reference invoked with a descriptive message if the pipeline
  140: encounters a character it cannot encode.  The function then C<die>s with
  141: a C<BUG:> prefix regardless.
  142: 
  143: =back
  144: 
  145: =head3 Returns
  146: 
  147: A defined scalar string containing only ASCII characters (code points 0x00â€“0x7F).
  148: 
  149: =head3 Side Effects
  150: 
  151: Prints diagnostic information to STDERR and invokes the C<complain> callback
  152: when an unhandled character is detected.  Dies with C<"BUG: wide_to_html(...)">
  153: in that case.
  154: 
  155: =head3 Usage Example
  156: 
  157:     use Encode::Wide qw(wide_to_html);
  158: 
  159:     my $safe = wide_to_html(string => "na\x{EF}ve caf\x{E9}");

  527: =head2 wide_to_xml
  528: 
  529: =head3 Purpose
  530: 
  531: Convert a Unicode / UTF-8 string into a pure-ASCII XML fragment by replacing
  532: every non-ASCII character with a hexadecimal numeric entity (e.g. C<&#x00E9;>).
  533: Unlike HTML, XML does not support most named entities, so only numeric forms
  534: are used.  Em-dashes and en-dashes are folded to a plain ASCII hyphen (C<->).
  535: Bare ampersands, angle brackets, and double-quotes are escaped for XML safety.
  536: 
  537: =head3 Arguments
  538: 
  539: Named parameters (or a single positional string as the first argument):
  540: 
  541: =over 4
  542: 
  543: =item C<string> (required)
  544: 
  545: The text to encode.  May be a plain scalar or a reference to a scalar.
  546: Must be defined; passing C<undef> causes the function to C<die>.
  547: 
  548: =item C<keep_hrefs> (optional, default 0)
  549: 
  550: When true, C<< < >>, C<< > >>, and C<< " >> are B<not> escaped.
  551: 
  552: =item C<complain> (optional)
  553: 
  554: A code reference invoked with a descriptive message if an unhandled character
  555: is encountered.  The function then C<die>s with a C<BUG:> prefix regardless.
  556: 
  557: =back
  558: 
  559: =head3 Returns
  560: 
  561: A defined scalar string containing only ASCII characters (code points 0x00â€“0x7F).
  562: 
  563: =head3 Side Effects
  564: 
  565: Prints diagnostic information to STDERR and invokes the C<complain> callback
  566: when an unhandled character is detected.  Dies with C<"BUG: wide_to_xml(...)">
  567: in that case.
  568: 
  569: =head3 Usage Example
  570: 
  571:     use Encode::Wide qw(wide_to_xml);
