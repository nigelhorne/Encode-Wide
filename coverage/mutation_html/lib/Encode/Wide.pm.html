  552: =head2 wide_to_xml
  553: 
  554: =head3 Purpose
  555: 
  556: Convert a Unicode / UTF-8 string into a pure-ASCII XML fragment by replacing
  557: every non-ASCII character with a hexadecimal numeric entity (e.g. C<&#x00E9;>).
  558: Unlike HTML, XML does not support most named entities, so only numeric forms
  559: are used.  Em-dashes and en-dashes are folded to a plain ASCII hyphen (C<->).
  560: Bare ampersands, angle brackets, and double-quotes are escaped for XML safety.
  561: 
  562: =head3 Arguments
  563: 
  564: Named parameters (or a single positional string as the first argument):
  565: 
  566: =over 4
  567: 
  568: =item C<string> (required)
  569: 
  570: The text to encode.  May be a plain scalar or a reference to a scalar.
  571: Must be defined; passing C<undef> causes the function to C<die>.
  572: 
  573: =item C<keep_hrefs> (optional, default 0)
  574: 
  575: When true, C<< < >>, C<< > >>, and C<< " >> are B<not> escaped.
  576: 
  577: =item C<complain> (optional)
  578: 
  579: A code reference invoked with a descriptive message if an unhandled character
  580: is encountered.  The function then C<die>s with a C<BUG:> prefix regardless.
  581: 
  582: =back

  584: =head3 Returns
  585: 
  586: A defined scalar string containing only ASCII characters (code points 0x00â€“0x7F).

  588: =head3 Side Effects

  591: detected, then dies with a C<BUG:> prefix regardless.

 1004: =head1 LIMITATIONS
 1005: 
 1006: =over 4
 1007: 
 1008: =item Character coverage is hand-maintained
 1009: 
 1010: Both functions use explicit byte_map tables.  Characters not listed fall back
 1011: to C<HTML::Entities::encode_entities_numeric> (HTML only) or trigger a fatal
 1012: C<BUG:> error (XML).  To add a missing character, extend the appropriate
 1013: C<@byte_map> array in both the UTF-8-bytes pass and the C<\N{U+...}> pass.
 1014: 
 1015: =item No C<< <script> >> / C<< <style> >> awareness
 1016: 
 1017: C<wide_to_html> applies entity escaping uniformly; it does not detect or skip
 1018: content inside C<< <script> >> or C<< <style> >> blocks.
 1019: Do not feed raw HTML documents through this function expecting semantic
 1020: preservation of embedded scripts.
 1021: 
 1022: =item XML numeric entity format
 1023: 
 1024: The XML numeric entities produced use a minimal hex representation
 1025: (e.g. C<&#x0E9;>) rather than the canonical 4-digit form (C<&#x00E9;>).
 1026: Both are valid XML 1.0, but consumers that do strict format matching should
 1027: normalise before comparing.
 1028: 
 1029: =item Input encoding assumption
 1030: 
 1031: The module decodes HTML entities first via C<HTML::Entities::decode>, then
 1032: applies byte-level substitutions.  Callers must not pass raw binary data
 1033: (non-UTF-8 byte strings) without first decoding them to Perl's internal
 1034: Unicode representation; undefined behaviour may result.
