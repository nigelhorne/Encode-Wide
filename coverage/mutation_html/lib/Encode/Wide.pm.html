  121: =head2 wide_to_html
  122: 
  123: =head3 Purpose
  124: 
  125: Convert a Unicode / UTF-8 string into a pure-ASCII HTML fragment by replacing
  126: every non-ASCII character with its named HTML entity (e.g. C<&eacute;>) where
  127: one exists, or a hexadecimal numeric entity (e.g. C<&#xNNNN;>) otherwise.
  128: Bare ampersands, angle brackets, and double-quotes are also escaped so the
  129: result is safe to embed in an HTML attribute or body.
  130: 
  131: =head3 Arguments
  132: 
  133: Named parameters (or a single positional string as the first argument):
  134: 
  135: =over 4
  136: 
  137: =item C<string> (required)
  138: 
  139: The text to encode.  May be a plain scalar or a reference to a scalar.
  140: Must be defined; passing C<undef> causes the function to C<die>.
  141: 
  142: =item C<keep_hrefs> (optional, default 0)
  143: 
  144: When true, C<< < >>, C<< > >>, and C<< " >> are B<not> escaped,

  146: 
  147: =item C<keep_apos> (optional, default 0)
  148: 
  149: When true, apostrophes (C<'>) are B<not> converted to C<&apos;>.
  150: Useful when the result will appear inside a JavaScript string literal.
  151: 
  152: =item C<complain> (optional)
  153: 
  154: A code reference invoked with a descriptive message if the pipeline
  155: encounters a character it cannot encode.  The function then C<die>s with
  156: a C<BUG:> prefix regardless.
  157: 
  158: =back
  159: 
  160: =head3 Returns
  161: 
  162: A defined scalar string containing only ASCII characters (code points 0x00â€“0x7F).
  163: 
  164: =head3 Side Effects
  165: 
  166: Invokes the C<complain> callback with a message when an unhandled character is

  538: =head2 wide_to_xml
  539: 
  540: =head3 Purpose
  541: 
  542: Convert a Unicode / UTF-8 string into a pure-ASCII XML fragment by replacing
  543: every non-ASCII character with a hexadecimal numeric entity (e.g. C<&#x00E9;>).
  544: Unlike HTML, XML does not support most named entities, so only numeric forms
  545: are used.  Em-dashes and en-dashes are folded to a plain ASCII hyphen (C<->).
  546: Bare ampersands, angle brackets, and double-quotes are escaped for XML safety.
  547: 
  548: =head3 Arguments
  549: 
  550: Named parameters (or a single positional string as the first argument):
  551: 
  552: =over 4
  553: 
  554: =item C<string> (required)
  555: 
  556: The text to encode.  May be a plain scalar or a reference to a scalar.
  557: Must be defined; passing C<undef> causes the function to C<die>.
  558: 
  559: =item C<keep_hrefs> (optional, default 0)
  560: 
  561: When true, C<< < >>, C<< > >>, and C<< " >> are B<not> escaped.
  562: 
  563: =item C<complain> (optional)
  564: 
  565: A code reference invoked with a descriptive message if an unhandled character
  566: is encountered.  The function then C<die>s with a C<BUG:> prefix regardless.
  567: 
  568: =back
  569: 
  570: =head3 Returns
  571: 
  572: A defined scalar string containing only ASCII characters (code points 0x00â€“0x7F).
  573: 
  574: =head3 Side Effects
  575: 
  576: Invokes the C<complain> callback with a message when an unhandled character is
  577: detected, then dies with a C<BUG:> prefix regardless.
  578: 
  579: =head3 Usage Example
  580: 
  581:     use Encode::Wide qw(wide_to_xml);
  582: 
  583:     my $safe = wide_to_xml(string => "SURN \x{017D}ganjar");

  984: =head1 LIMITATIONS
  985: 
  986: =over 4
  987: 
  988: =item Character coverage is hand-maintained
  989: 
  990: Both functions use explicit byte_map tables.  Characters not listed fall back
  991: to C<HTML::Entities::encode_entities_numeric> (HTML only) or trigger a fatal
  992: C<BUG:> error (XML).  To add a missing character, extend the appropriate
  993: C<@byte_map> array in both the UTF-8-bytes pass and the C<\N{U+...}> pass.
  994: 
  995: =item No C<< <script> >> / C<< <style> >> awareness
  996: 
  997: C<wide_to_html> applies entity escaping uniformly; it does not detect or skip
  998: content inside C<< <script> >> or C<< <style> >> blocks.
  999: Do not feed raw HTML documents through this function expecting semantic
 1000: preservation of embedded scripts.
 1001: 
 1002: =item XML numeric entity format
 1003: 
 1004: The XML numeric entities produced use a minimal hex representation
 1005: (e.g. C<&#x0E9;>) rather than the canonical 4-digit form (C<&#x00E9;>).
 1006: Both are valid XML 1.0, but consumers that do strict format matching should
 1007: normalise before comparing.
 1008: 
 1009: =item Input encoding assumption
 1010: 
 1011: The module decodes HTML entities first via C<HTML::Entities::decode>, then
 1012: applies byte-level substitutions.  Callers must not pass raw binary data
 1013: (non-UTF-8 byte strings) without first decoding them to Perl's internal
 1014: Unicode representation; undefined behaviour may result.
 1015:
